## 修订任务

### 修订 1：新增交互场景——鼠标悬停"九宫格透显"效果

**交互需求描述**：
- 默认状态下，渔网网格在地图上**不可见**（完全透明）
- 用户鼠标在地图上移动时：
  - 鼠标所在位置的渔网单元格（中心格）半透明显示，**有清晰边框**
  - 中心格的**周围八个格子**（共 3×3 九宫格）也同时显示，但**透明度更高**（更淡）
  - 视觉重点为中心格，外围八格作为方位参考
- 鼠标移动时，九宫格效果跟随鼠标实时刷新
- 鼠标离开地图区域时，九宫格效果消失

**视觉参考**：
```
┌──┬──┬──┐
│░░│░░│░░│   外围八格（低透明度，淡）
├──┼──┼──┤
│░░│██│░░│   中心格（中等透明度，鼠标所在位置，清晰边框）
├──┼──┼──┤
│░░│░░│░░│
└──┴──┴──┘
    ↑ 鼠标所在位置
```

**PRD 中需要新增/更新的内容**：
- 在功能需求"地图面板"章节新增"鼠标悬停九宫格透显"小节
- 描述触发条件、视觉效果、性能要求（鼠标移动跟随刷新无明显卡顿）
- 验收标准新增："鼠标在地图上移动时，九宫格透显效果跟随刷新延迟 < 100ms"

**端差异说明**：此功能仅政府端可见（企业端无渔网层）

### 修订 3：EvoMap 展示视角——从开发者视角切换为用户视角

**当前问题**：PRD 中 EvoMap 状态面板暴露了过多后台信息（节点状态、已发布 Capsule 数量、心跳时间等），这些是开发者/Orchestrator 关心的内容，**普通用户不需要看到**。

**修订要求**：
- 用户视角只展示"自进化效果"，**不暴露后台技术细节**
- 用户可见的展示内容（举例，由内容 Agent 提炼）：
  - "本次评估已学习了 N 次历史决策经验"
  - "Agent 当前对桐庐县产业偏好的理解程度：XX%"
  - "已遗传/共享的评估方法论：X 条"
  - 或图形化展示进化曲线/能力雷达图（让用户直观感受 Agent "变聪明了"）
- **隐藏**的开发者信息（移到后台日志或开发者调试面板）：
  - 节点 ID / node_secret / 心跳时间戳
  - Capsule 原始 JSON / asset_id / 发布门槛数值
  - EvoMap Hub URL / 协议版本号

**PRD 中需要更新的内容**：
- 功能需求"状态面板"改为"自进化展示面板"
- 列出用户可见字段（简洁、形象、有故事感）
- 标注"后台调试信息"独立列表，作为开发参考但不出现在用户界面

### 修订 4：地图边界硬约束——不超出桐庐县外接矩形

**需求描述**：
- 在线地图使用**天地图（中国官方）**，主人已申请个人开发者 API
- 用户可缩放地图尺度级别（如 z10 → z18）
- 但用户视野的矩形地理范围**不得超出桐庐县行政边界的外接矩形**（最大外接矩形 = min_lng, min_lat, max_lng, max_lat 围成的矩形）
- 当用户尝试拖动地图或缩小到外接矩形之外时，地图应自动回弹到边界内

**桐庐县外接矩形参考**：
- 大致经度范围：119.16°E – 119.80°E
- 大致纬度范围：29.58°N – 30.12°N
- 实际值由地理数据精确推算（架构/实现阶段处理）

**PRD 中需要更新的内容**：
- 在功能需求"地图面板"章节新增"地图视野边界约束"小节
- 底图来源更新为"天地图（API Key 由主人提供）"，离线瓦片作为兜底备份
- 验收标准新增：
  - "用户拖动地图至外接矩形外，地图自动回弹"
  - "地图缩放级别 z10-z18，超出范围禁用缩放按钮"
