## 修订 1：行政区划边界悬停高亮

### 用户原始需求

> 将行政区矢量边界显示出来，但不是直接显示。如果鼠标放在某个行政区上，该行政区的边界线亮起来展现出来。

### 结构化翻译（PM 语言）

**功能名称**：行政区划悬停高亮

**触发条件**：
- 用户在**政府端**地图上移动鼠标，光标停留在任意乡镇/街道辖区范围内
- 鼠标移出该辖区范围时效果消失

**视觉规格**：
- 默认状态：14 个乡镇/街道边界**不可见**（透明）
- 悬停状态：光标所在乡镇/街道的多边形边界以**亮色线段**（建议 `#f0883e` 或 `#58a6ff`，2px 宽）高亮绘制
- 过渡：无动画，直接绘制/移除（降低实现复杂度）
- 叠加 Z 序：在渔网层之上、标注层之下

**数据来源**：`data/processed/XZQ.geojson`（14 个乡镇行政边界，WGS84，已可用）

**技术约束**：
- 纯前端实现，不新增 API
- XZQ.geojson 约 5MB，前端 fetch 一次后缓存在内存
- 使用 Leaflet `L.geoJSON` 的 `onEachFeature` 绑定 mouseover/mouseout 事件
- 性能：14 个 Polygon 的 `point-in-polygon` 判断在 mousemove 中执行，需 ≤ 80ms throttle

**验收标准**：
- 鼠标滑入富春江镇 → 富春江镇边界亮起橙色线 → 滑出 → 线消失
- 鼠标在两个乡镇交界处 → 当前所在乡镇高亮，邻镇不高亮
- 企业端不触发此效果
- 不遮挡九宫格和框选矩形

## 修订 2：多级渔网空间聚合金字塔

### 用户原始需求

> 不同缩放尺度展示不同尺度的渔网。事先准备8个级别的渔网，镜头拉最远时展示尺度最大的渔网。用户在不同缩放级别，后台拉取不同尺度的渔网。

### 结构化翻译（PM 语言）

**功能名称**：渔网 LOD（Level of Detail）金字塔

**设计目标**：
- 解决"全县鸟瞰时 100m 网格太密→框选数据暴增→渲染卡顿"问题
- 不同 zoom 级别自动切换对应聚合层级的网格数据

**8 级金字塔规格**：

| Zoom | 网格边长 | 合并比 | 预计算方式 | 九宫格行为 |
|:--:|------|:--:|------|------|
| 11 | 3,200m | 32×32→1 | 32×32 渔网空间聚合 | 单格悬停（网格太大，不展示 3×3） |
| 12 | 1,600m | 16×16→1 | 16×16 聚合 | 单格悬停 |
| 13 | 800m | 8×8→1 | 8×8 聚合 | 3×3 九宫格 |
| 14 | 400m | 4×4→1 | 4×4 聚合 | 3×3 九宫格 |
| 15 | 200m | 2×2→1 | 2×2 聚合 | 3×3 九宫格 |
| 16 | 100m | 原始 | 无需聚合 | 3×3 九宫格 |
| 17 | 100m | 原始 | 无需聚合 | 3×3 九宫格 |
| 18 | 100m | 原始 | 无需聚合 | 3×3 九宫格 |

**聚合网格属性**（每格存储）：

| 字段 | 来源 | 说明 |
|------|------|------|
| `grid_id` | `grid_L{level}_{r}_{c}` | 层级+行列号 |
| `land_type` | 被合并网格的地类众数（出现最多的 DLMC） | 主导地类 |
| `land_type_top3` | JSON `[{"name":"乔木林地","pct":0.65}, ...]` | 占比前三地类及比例 |
| `land_code` | 众数地类对应的 DLBM | 用于兼容现有 schema |
| `area_sqm` | 合并网格总面积（100m×合并边长的平方） | 如 800m 网格 = 640,000 m² |
| `ownership` | "混合（N 单位）" | 简化权属表示 |
| `town` | 被合并网格的坐落众数 | 主导乡镇 |
| `mixed_type` | 如果单一地类占比<70%则为混合，否则 NULL | 混合标识 |
| `min_lng/min_lat/max_lng/max_lat` | 聚合网格外接矩形 | WGS84 bbox |
| `geometry` | 外接矩形 WKB | 简化几何 |
| `grid_count_original` | 被合并的原始网格数 | 用于统计 |
| `level` | 金字塔层级 1-8 | zoom 级映射 |

**后端改动**：
- 新增 `scripts/preprocess_fishnet_pyramid.py`：从 100m 原始数据生成 5 级聚合表
- 数据库新增 5 张表：`land_grid_L1`(3200m) ~ `land_grid_L5`(200m)，原始表为 `land_grid_L0`
- `grid_service.py` 的 `query_by_bbox()` 和 `query_nine_grid()` 根据 zoom 参数选择对应表
- API 入参新增 `zoom: int = Query(12)`

**前端改动**：
- 框选/九宫格请求携带当前 `zoom` 参数
- 九宫格在 zoom 11-12 时仅渲染单格（中心格），zoom 13+ 渲染 3×3

**性能指标**：
- 任意 zoom 级框选响应 < 200ms
- zoom 11 全县框选网格数 < 500，无需截断
- 前端 GeoJSON 渲染 < 100ms

**降级策略**：
- 聚合表未生成时，zoom 11-15 降级到 100m 原始数据（加截断）
