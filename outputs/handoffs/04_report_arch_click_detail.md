## 架构汇报（点选详情面板 UI 设计）

**任务**：设计地图点选详情面板的 UI 布局，对比 3 个方案，输出推荐方案及完整 HTML/CSS/交互流程。

**完成状态**：完成

**架构文档路径**：`specs/arch/click_detail_ui_design.md`

---

**核心设计决策**：

1. **推荐方案 B（浮动叠加层）** — 详情卡片 + 半透明遮罩覆盖在标签页内容上方，不中断用户当前的聊天/企业列表上下文。关闭后一切恢复如初，零状态丢失。对标 Google Maps 地点详情、Figma 属性面板的交互模式。

2. **方案 A（第四标签页）淘汰理由**：点选详情是"看一眼就关"的瞬时操作，切换整页标签过于笨重，且需要记录返回目标标签

3. **方案 C（替换内容区）淘汰理由**：需要 store/restore 滚动位置和 DOM 状态，标签切换后"返回"语义模糊

4. **关键 DOM 变更**：新增 `#panel-overlay`（遮罩层）+ 改造现有 `#detail-panel` 为绝对定位卡片。无需删除现有节点，仅修改 CSS `position` 和新增 HTML 元素

5. **四种关闭方式**：× 按钮、点击遮罩、点击地图空白、Esc 键——覆盖全部用户直觉

6. **标签头不遮罩**：用户可以在详情打开时自由切换标签页（如查看自进化面板对比），这是有意保留的交互自由度

---

**风险提示**：无。方案 B 是成熟 UI 模式，不引入新的布局风险。

**待确认事项**：无。
