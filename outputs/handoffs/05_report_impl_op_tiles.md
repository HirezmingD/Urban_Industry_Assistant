## 根因

天地图 key `48ad1b764d4d7381c3d16c84848222c4` 为**浏览器端类型**（决议 4 明确标注），服务器端 HTTP 请求（`requests.get`）缺少浏览器 Referer 头，天地图服务端拒绝返回瓦片。

**这不是脚本 bug**，是 key 类型决定的。浏览器端 key 只能通过浏览器请求（Leaflet `L.tileLayer` 自动带 Referer），无法用于脚本批量下载。

## 建议

1. **在线模式优先**：前端 Leaflet `L.tileLayer` 加载在线瓦片完全正常（已验证），演示现场有网络即可
2. **离线兜底保留**：前端 `errorTileUrl: '/data/tiles/{z}/{x}/{y}.png'` 兜底代码已在，如果现场网络异常，前端优雅降级（瓦片加载失败 → 显示空白/灰色 tile，不会崩溃）
3. **如需离线**：需申请天地图服务端 key（而非浏览器端 key），或手动用浏览器逐个下载（不现实）
