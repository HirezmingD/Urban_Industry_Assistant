# 开发流程方案：Urban_Industry_Assistant

> **受众**：开发团队内部
> **目标**：19 号中午 → 21 号中午，48 小时内完成可演示的网页端 Demo
> **实际开发时间**：约 20 小时（扣除吃饭睡觉）

---

## 整体架构

```
浏览器（评委）
    │
    ├─ 地图面板（Leaflet.js）       ← 框选、点选、GeoJSON 渲染
    ├─ 对话面板（聊天界面）          ← 输入需求、查看 Agent 回复
    └─ 状态面板（EvoMap 进化状态）   ← 展示 Gene/Capsule 发布记录
            │
    HTTP (JSON)
            │
    服务器（FastAPI，2核2G）
    ├── 地图 API        /api/map/*       ← 三调数据查询、GeoJSON 返回
    ├── Agent API       /api/agent/*     ← LLM 对话、产业评估
    ├── 企业 API        /api/enterprise/*← 企业需求匹配
    └── EvoMap API      /api/evomap/*    ← hello/publish/report
            │
    本地文件
    ├── data/           ← 三调 GeoJSON、卫星图瓦片、栅格数据
    └── db.sqlite       ← 企业画像、评估日志、用户偏好
```

## 阶段一：后端骨架（19 号中午 - 19 号傍晚）

### 目标
地图数据 API + Agent 对话 API 能跑通。

### 任务清单

| # | 任务 | 细节 | 产出 |
|---|------|------|------|
| 1.1 | **项目初始化** | FastAPI 项目结构、依赖安装 | `main.py` + `requirements.txt` |
| 1.2 | **地图数据 API** | 加载三调 GeoJSON，提供范围查询接口 | `GET /api/map/query?bbox=...` 返回范围内的图斑 |
| 1.3 | **图斑详情 API** | 根据点击坐标返回单块地属性 | `GET /api/map/parcel?lat=...&lng=...` 返回面积/性质/权属 |
| 1.4 | **LLM 对话 API** | 接入 LLM API，构建 prompt 模板 | `POST /api/agent/chat` 返回评估+建议 |
| 1.5 | **SQLite 初始化** | 建表（评估日志、企业画像、用户偏好） | `db.sqlite` |

### 依赖清单
```
fastapi
uvicorn
geopandas
shapely
requests        # 调 LLM API + EvoMap
sqlite3         # 内建
```

### 验证
```bash
# 用 curl 测试
curl "http://localhost:8000/api/map/query?bbox=119.5,29.5,119.8,29.8"
curl -X POST "http://localhost:8000/api/agent/chat" -d '{"message":"城东30亩地适合什么产业"}'
```
两条都返回有效 JSON 即通过。

## 阶段三：评估逻辑与企业匹配（20 号上午 - 20 号下午）

### 目标
Agent 不是泛泛聊天，产出有针对性的产业评估。

### 任务清单

| # | 任务 | 细节 | 产出 |
|---|------|------|------|
| 3.1 | **评估 Prompt 工程** | 构建产业评估的 system prompt，注入数据上下文 | 评估质量提升 |
| 3.2 | **评估结果结构化** | 输出格式：评分 + 适配产业 + 理由 + 风险提示 | JSON Schema |
| 3.3 | **企业数据入库** | 虚构企业 case 写入 SQLite | 5-8 条企业记录 |
| 3.4 | **企业匹配 API** | 根据企业需求匹配可用地块 | `GET /api/enterprise/match?need=...` |
| 3.5 | **用地绩效数据入库** | 同伴计算好的指标写入 SQLite，供前端调用 | 指标数据就位 |

### Prompt 模板参考
```
你是一个县域产业用地评估专家。你的知识范围是浙江省桐庐县。
当前评估地块信息：
- 面积：{area}
- 当前用地性质：{land_type}
- 权属：{ownership}
- 周边业态：{surrounding}

请给出：
1. 产业适配评分（1-10分）
2. 推荐产业类型（3个，排序）
3. 每个推荐的理由
4. 风险提示
```

## 阶段四：EvoMap 集成（20 号下午）

### 目标
Agent 注册 EvoMap 节点，发布至少一个进化胶囊。

### 任务清单

| # | 任务 | 细节 | 产出 |
|---|------|------|------|
| 4.1 | **注册节点** | POST /a2a/hello，获取 node_secret | 节点上线 |
| 4.2 | **心跳保活** | 每 5 分钟发一次 heartbeat | 节点保持 alive |
| 4.3 | **构造 Gene** | 评估方法论打包为 Gene（如"产业集中度指标纳入评估"） | Gene JSON |
| 4.4 | **构造 Capsule** | 一次成功评估打包为 Capsule（附带验证结果） | Capsule JSON |
| 4.5 | **发布捆绑包** | POST /a2a/publish，Gene + Capsule 一起发 | 资产上链 |
| 4.6 | **前端状态面板** | 展示节点状态、已发布 Capsule 数量、GDI 评分 | 评委可视化看到进化 |

### 自进化演示剧本（录屏备选）
如果实时跑不完，用录屏替代：

```
第一轮：评估城东 30 亩 → 推荐"轻工业" → 用户否定"服装业饱和"
```
