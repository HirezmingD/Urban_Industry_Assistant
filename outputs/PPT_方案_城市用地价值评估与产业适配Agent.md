# PPT 制作方案：城市用地价值评估与产业适配智能体

> **目标听众**：对项目背景和技术细节不了解的外部听众（评委、投资人、合作方）
> **时长**：8-10 分钟路演（约 15-18 页）
> **风格**：简洁、专业，每页不超过 3 个核心信息点，正文建议 14pt+，少字多图

---

## 整体叙事线

```
痛点引入 → 为什么是现在 → 我们做什么 → 怎么做到 → 
为什么能"进化" → 数据安全怎么保证 → Demo 怎么跑 → 价值与下一步
```

## 逐页方案

### 第 1 页 | 封面

**页面内容**：
- 项目名称：Urban_Industry_Assistant — 城市产业空间智能助手
- 副标题：为县域现代化产业体系提供 AI 驱动的空间决策咨询
- 赛道：The Pearl · 智能体自进化
- 团队/作者信息

**制作者备注**：保持干净、留白多，项目名用大号字体。可用一张抽象的城市/地图线条作为背景。

---

### 第 2 页 | 一个真实的问题

**页面内容**（场景化引入）：
- 标题：一个中小企业主的烦恼
- 正文（讲故事，3 句话）：
  > "我需要 2000 平的厂房，在城东。但不知道哪里有地、地能不能用、政府会不会批。"
- 数字强化：
  - 全国中小企业超 5200 万家
  - 县域产业用地供需匹配效率不足 30%（估算，可标注"行业观察"）

**制作者备注**：用人物剪影 + 问号图标，共情式开场。不要直接丢"项目背景"四个字。

---

### 第 3 页 | 三块拼图，各自为战

**页面内容**：
- 标题：三块拼图，中间缺了一个协调者
- 三角布局（三张卡片，等距排列）：
  - 🏛️ **政府** — 一级土地主导者，追求产业升级、税收、就业、土地利用效率
  - 🏢 **大企业** — 闲置用地持有者，追求资产盘活、变现或合作开发
  - 🏭 **中小企业** — 零星用地需求方，追求找到合适的生产空间
- 三角中心（缺口标记）：三方利益有交集但不重叠——缺乏一个同时理解产业逻辑、土地规则、多方诉求的协调角色
- 底部标注：目标一致（双赢），立场各不同——这不是"匹配"，是多方博弈下的最优解

**制作者备注**：用三角形布局，三个角色各占一角，中心放缺口图标。体现三方关系的张力与交集。

---

### 第 4 页 | 不只是用地——更大的图景

**页面内容**：
- 标题：县域经济的三道坎
- 图标 + 关键词形式：
  - 🔗 产业链不完整 — 缺链、断链、低端锁定
  - ⚡ 创新链薄弱 — 科创资源向县域传导不畅
  - 🔀 要素错配 — 有地的不需要，需要的找不到
- 一句话总结：**用地问题只是表象，本质是产业体系与空间要素的适配失效**

**制作者备注**：这一页把格局从"找地"提升到"产业体系优化"，为后续Agent的定位做铺垫。

---

### 第 5 页 | 我们要做什么

**页面内容**：
- 标题：Urban_Industry_Assistant
- 一句话定位：**面向县域政府的产业用地智能评估与适配引擎**
- 三大能力（图标 + 一行说明）：
  - 🗺️ **空间评估** — 任意框选地块，输出产业适配评分
  - 🔍 **需求匹配** — 企业需求 ↔ 可用地块的智能对接
  - 🧠 **自进化** — 每次交互都在变聪明，经验可传承

**制作者备注**：这是全PPT最核心的一页，建议用简洁的图标矩阵，不要文字堆砌。每个能力配一个具象的 mini 示例。

---

### 第 6 页 | 场景演示（政府端）

**页面内容**：
- 标题：政府人员的一天
- 场景化步骤（1-2-3）：
  1. 在地图上框选城东 30 亩地块
  2. Agent 输出：地块属性 + 周边可用地 + 产业适配建议（"该区域物流基建评分高，建议精密制造/仓储"）
  3. 政府人员反馈："服装业不考虑"→ Agent 记住偏好，下次自动排除
- 底部标注：Demo 以浙江省桐庐县为样本区域

**制作者备注**：如果 PPT 能嵌入动图/GIF，展示地图交互过程效果最佳。否则用三帧静态截图并排。

---

### 第 7 页 | 场景演示（企业端 — 轻量）

**页面内容**：
- 标题：企业的一句话需求
- 交互演示：
  - 输入："我要 1000 平，做食品加工，在分水镇附近"
  - Agent 输出：匹配列表（方位、面积、性质、权属、评分）+ 适配建议
- 底部标注：企业端定位为轻量的需求画像与匹配，决策仍以政府端为核心

**制作者备注**：一个界面截图即可，展示输入-输出流程。

---

### 第 8 页 | 为什么能"自进化"？

**页面内容**：
- 标题：一个越来越聪明的 Agent
- **三轮进化剧本**（竖状时间线）：

| 交互轮次 | 用户反馈 | Agent 的变化 |
|---------|---------|-------------|
| 第 1 轮 | "城东已经有五个服装厂了" | 学会排除饱和产业 |
| 第 5 轮 | 持续采纳物流类建议 | 自动提升物流适配权重 |
| 第 N 轮 | 另一个用户评估类似地块 | 继承前人经验，跳过低效推荐 |

- 关键词：**失败经验 → 策略修正 → 跨用户遗传**

**制作者备注**：这一页要让人一眼看懂"进化"。建议用时间线 + 箭头动效，或三列并排对比图。

---

### 第 9 页 | 自进化的技术底座 —— EvoMap

**页面内容**：
- 标题：EvoMap — Agent 的"DNA 遗传系统"
- 一句话解释：像 Git + GitHub，但传输的不是代码，是**经过验证的智力经验**
- 三层结构图：
  - 🧬 **Gene**（策略基因）— "产业集中度应该纳入评估指标"
  - 💊 **Capsule**（经验胶囊）— "该策略在桐庐城东验证有效，置信度 0.92"
  - 🔄 **Hub 自然选择** — 高评分的胶囊被自动推广，低分的被淘汰

**制作者备注**：用 Git ↔ GitHub、Gene ↔ Capsule 的类比让技术概念落地。图示最关键。

---

### 第 10 页 | 敏感数据怎么办？

**页面内容**：
- 标题：公私分离 —— 决策在本地，方法论上 Hub
- 左右分栏：
  - 🏠 **本地（绝不外传）**：地块坐标、评估结论、企业信息、用户偏好
  - ☁️ **Hub（脱敏公开）**：评估方法论模板、信号匹配规则、通用验证模式
- 一句话总结：**自进化分两条线——纵向（自身越用越聪明）+ 横向（跨 Agent 经验遗传），互不污染**

**制作者备注**：这一页很重要，评委/合作方会有数据安全疑虑。用锁/盾牌图标强化安全感。如果 EvoMap 文档中有脱敏规则的截图，可以用。

### 第 11 页 | 技术架构总览

**页面内容**：
- 标题：技术架构
- 架构图（分层）：
  - 🖥 **交互层** — 对话式界面 + 地图组件（框选/点选）
  - 🧠 **智能体层** — LLM 推理 + 工具调用 + 记忆系统
  - 📊 **数据层** — 三调矢量数据 + 房价爬取 + 开源产业数据
  - 🧬 **进化层** — EvoMap/Evolver（Gene/Capsule + Hub 遗传）
- 技术栈标注（Python / GIS / EvoMap / LLM API）

**制作者备注**：架构图用分层方框图，不要用复杂 UML。颜色区分四层。
